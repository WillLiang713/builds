# ImmortalWrt Builder

This repository provides one generic builder for MediaTek Filogic firmware.
Normal use happens from the repository root.

## Quick Start

Run from the repository root:

```bash
make
```

On first use, the menu automatically prepares the upstream source tree and
feeds. Then use the menu flow:

```text
1. Select device profile
2. Select optional packages
3. Build firmware
```

If you skip optional package selection, the build uses only the default package
configuration.

## Menu

```text
1. Select device profile
2. Select optional packages
3. Build firmware
4. Show current configuration
5. Update upstream source and feeds
6. Clean build cache
7. Open builder shell
0. Exit
```

There is no strict required order, but the usual flow is:

```text
select device -> optional packages -> build firmware
```

If you start a build before selecting a device, the script opens device
selection first. Device profiles are parsed from the current upstream file:

```text
source/target/linux/mediatek/image/filogic.mk
```

## Configuration

`.env` stores the current selection and local overrides. You can create it from
the example file:

```bash
cp .env.example .env
```

Common variables:

```text
DEVICE_PROFILE=             selected device profile, written by the menu
UPSTREAM_REPO=              upstream source repository
UPSTREAM_REF=               branch, tag, or commit; empty uses the default branch
BASE_DEFCONFIG=             manual base defconfig override; usually empty
JOBS=                       build job count; empty uses nproc
BUILD_VERBOSE=0             set to 1 for verbose build logs
```

Default package configuration lives in:

```text
configs/default.seed
```

Device-specific additions can be placed in:

```text
configs/devices/<DEVICE_PROFILE>.seed
```

Packages selected from the menu are written to:

```text
configs/custom.seed
```

`configs/custom.seed` is ignored by git and is intended for local temporary
package choices.

## Build Logic

The OpenWrt `.config` is generated by merging:

```text
automatically selected or manually configured BASE_DEFCONFIG
selected Filogic device profile
configs/default.seed
configs/devices/<DEVICE_PROFILE>.seed, if present
configs/custom.seed, if present
```

If the selected device has no matching local base defconfig, the build stops and
asks you to set `BASE_DEFCONFIG` in `.env`.

If upstream includes bundled archives under `source/dl`, the scripts copy them
to the external `dl` cache used by the build. This avoids unnecessary downloads
and reduces network-related build failures.

## Output

Built firmware is copied to:

```text
output/<DEVICE_PROFILE>/<date>-<upstream-commit>/
```

## Advanced Commands

The recommended daily entry point is:

```bash
make
```

When you need to run a specific step directly:

```bash
make init        # update source tree and feeds
make device      # select a Filogic device profile
make packages    # select optional firmware packages
make build       # build firmware for the selected device
make menuconfig  # open OpenWrt menuconfig
make clean       # run OpenWrt clean
make distclean   # remove generated build state under source/
make shell       # open a shell in the builder container
```

The old `make select-device` and `make select-packages` commands have been
removed. Use `make device` and `make packages` instead.
